LegacyDb2SystemDescr 1.8.70

Extension of J1939-22

  • Container mapping of requestable J1939-22 messages is improved reflecting the possibility, to receive and transmit requestable PDU1 PGs with FEFF (standard Id), as well as with FBFF (extended Id; if globally addressed).
  • Admin Data is derived for proprietary J1939TpConnection and J1939TpPG parameters which aren`t yet supported by AUTOSAR
  • Support of CAN FD Basic Frames (FBFF)
  • Set Priority of MultiPg to highest C-PG Priority
  • Remove legacy parameter J1939PgAppearanceOnBus which was used for determination of J1939-22 PDU
  • Exclude J1939-22 PGs from container which shall appear on the bus as dedicated FEFFs, not as C-PGs
  • Improve send behaviour for cyclic non-requestable J1939-22 DBC message removing the timeout and setting the trigger to ALWAYS

Miscellaneous Features

  • Check for inconsistent NID / CBV position of NM messages and warn the user if an inconsistent signal layout is detected

Fixed Issues

  • Attribute for usage of MultiPg is not derived correctly from DBC and therefore not detected by the DaVinci Tooling during import
  • J1939-22 requestable messages are incorrectly derived in case GenMsgSendType is set to cyclic and in destination address of the DBC message is not equal to 0xFE (Null Address)

LegacyDb2SystemDescr 1.8.69

Miscellaneous Features

  • Derive E2E Offset for Assurance Data, also covering Signal Groups

Fixed Issues

  • IpduMContainerTxFirstContainedPduTrigger shall not be set to true for J1939-22 related containers
  • CanNm Control Bit Vector and Source Node Identifier detection does not work as expected and is not described in the Technical Reference
  • Missing DataTransformationSet in case of a dedicated TransformationTechnology for each E2E protected ISignal is necessary

(top)

LegacyDb2SystemDescr 1.8.68

Miscellaneous Features

  • Derive E2E Offset for Assurance Data
  • Derive correct container short headers (TOS, TF) instead of TOS and TF being derived each as constant 000b
  • Remove creation of IpduMContainedTxPduSendTimeout at J1939-22 CANFD
  • Don't create containers for J1939-22 on CAN FD if J1939-21 is specified via SAE_J1939_21_SpecVersion
  • Introduce attribute 'ReRoutJ1939ToCom'

Fixed Issues

  • E2E protection related SignalGroups with TX/RX Signals are mixed together
  • Wrong TOS are derived in Contained PDU Headers being constantly derived to 000b
  • Several ContainedIPdu within one ContainerIPdu lead to unsuccessful conversion

(top)

LegacyDb2SystemDescr 1.8.67

Miscellaneous Features

  • Support of user data in NM message
  • Support of SecuredIPdu for J1939-22

Fixed Issues

  • Value of IpduMContainedTxPduSendTimeout out of range
  • Partially incorrect derivation of J1939-22 pdu with IpduMContainedTxPduCollectionSemantics derived as IPDUM_COLLECT_QUEUED instead of IPDUM_COLLECT_LAST_IS_BEST and IpduMContainedTxPduTrigger derived as IPDUM_TRIGGER_NEVER instead of IPDUM_TRIGGER_ALWAYS
