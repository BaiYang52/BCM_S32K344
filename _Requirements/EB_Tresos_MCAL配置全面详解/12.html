第12章 串行外设接口（Spi）：Spi模块概述、主从模式配置、数据帧格式、Job与Sequence配置

好，咱们今天聊聊SPI。说实话，在汽车嵌入式里，SPI是个老面孔了。从传感器到执行器，从EEPROM到看门狗，到处都有它的身影。我最早接触SPI是在做车身控制模块的时候，那时候被一个片选信号的毛刺折腾了整整两天——嗯，这个坑我后面会细说。

12.1 Spi模块概述

SPI，全称Serial Peripheral Interface，说白了就是同步串行通信。它用四根线搞定一切：SCK（时钟）、MOSI（主出从入）、MISO（主入从出）、CS（片选）。

在AUTOSAR架构里，Spi模块属于MCAL层。它负责管理底层的SPI硬件，同时向上层提供统一的接口。我个人习惯把Spi模块看作一个「交通调度员」——它不关心数据是什么，只关心数据怎么走、什么时候走。

核心要点：

  • Spi模块支持同步和异步传输
  • 支持DMA和中断模式
  • 可以配置多个通道（Channel），每个通道对应一个片选信号
  • 支持Job和Sequence的层级管理

你可能会问：「为什么不用I2C？」我在项目里遇到过类似的问题。SPI的优势在于速度快、全双工、协议简单。缺点嘛，就是线多，而且没有标准的应答机制。所以选型的时候，你得想清楚：是要速度，还是要可靠性？

12.2 主从模式配置

SPI的通信模式很简单：一个主设备，一个或多个从设备。主设备负责产生时钟信号，从设备被动响应。

在EB Tresos里配置主从模式，主要关注这几个参数：

参数 说明 我的建议
SpiMaster 配置为主模式还是从模式 大部分场景用主模式，从模式少用
SpiClockPolarity 时钟极性（空闲时高电平还是低电平） 看从设备手册，别猜
SpiClockPhase 时钟相位（数据在上升沿还是下降沿采样） 这个配错了，数据全乱
SpiDataWidth 数据位宽，通常是8位或16位 8位最通用

注意：我曾经遇到过一个问题：主设备和从设备的时钟极性和相位配置不一致，结果通信时好时坏。后来用示波器一抓，才发现是CPOL和CPHA搞反了。所以，配置之前一定要仔细看从设备的datasheet。

12.3 数据帧格式

SPI的数据帧格式，说白了就是数据怎么打包、怎么传输。在AUTOSAR的Spi模块里，数据帧格式由以下几个要素决定：

  • 数据长度：每个数据帧有多少位
  • 传输顺序：MSB first还是LSB first
  • 片选行为：传输前后片选信号如何变化

举个例子，假设你要读写一个SPI Flash芯片：

/* 伪代码示例：SPI Flash读操作 */
Spi_WriteIB(SpiChannel_Flash, &cmd_buffer, 1);  // 发送读命令
Spi_ReadIB(SpiChannel_Flash, &data_buffer, 4);  // 读取4字节数据
Spi_AsyncTransmit();  // 启动传输

这里要注意的是，很多SPI设备要求片选信号在整条命令+数据期间保持低电平。如果你把片选拆成了两次传输，设备就会认为命令结束了——嗯，这个坑我踩过。

小技巧：对于需要连续片选的场景，可以使用AUTOSAR的SpiSequence功能，把多个Job打包成一个Sequence，保证片选信号不中断。

12.4 Job与Sequence配置

这是AUTOSAR Spi模块的精髓所在。Job和Sequence的概念，说白了就是「任务」和「任务组」。

12.4.1 Job是什么？

一个Job代表一次完整的SPI传输。它包含：

  • 一个或多个Channel（通道）
  • 每个Channel对应的发送缓冲区和接收缓冲区
  • 传输完成后的回调函数（可选）

我习惯把Job理解成「一次对话」。比如你要读一个传感器的数据，这就是一个Job。

12.4.2 Sequence是什么？

Sequence是多个Job的集合。它们按顺序执行，中间不会被打断。这在需要连续操作多个设备时特别有用。

举个例子：

/* Sequence配置示例 */
Sequence_1:
  Job_1: 写命令到设备A
  Job_2: 读数据从设备A
  Job_3: 写命令到设备B
  Job_4: 读数据从设备B

在EB Tresos里配置Sequence时，有几个关键参数：

参数 说明
SpiSequenceJobList 该Sequence包含哪些Job，按顺序排列
SpiSequenceJobCount Job的数量
SpiSequenceType 同步还是异步执行

重点：Sequence的执行是原子的。也就是说，一旦开始执行一个Sequence，除非发生错误，否则不会中断。这保证了多设备操作的时序一致性。

12.4.3 配置时的注意事项

我在项目中遇到过一个问题：某个Sequence里有5个Job，但第3个Job的执行时间特别长，导致后面的Job超时。后来发现是那个Job的缓冲区大小配置错了，数据量比预期大了一倍。

所以，配置Job和Sequence时，我建议你：

  1. 先搞清楚每个Job的数据量
  2. 估算每个Job的执行时间
  3. 考虑是否需要中断或DMA
  4. 测试时用示波器抓一下实际波形

避坑指南：我曾经在配置Sequence时，把两个Job的片选信号搞混了。结果设备A的片选信号一直拉低，设备B的片选信号一直拉高，通信完全乱套。检查了半天才发现是Job和Channel的映射关系配错了。

12.5 小结

SPI模块的配置，说难不难，说简单也不简单。关键是要理解主从模式、数据帧格式、Job和Sequence这几个核心概念。我个人觉得，只要把Job和Sequence的关系理清了，剩下的就是参数填表的事。
