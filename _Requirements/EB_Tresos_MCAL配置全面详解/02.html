第二课：工作区与项目创建

大家好，欢迎来到第二课。今天咱们聊聊EB Tresos里最基础、但也最容易踩坑的部分——工作区和项目创建。

说实话，我见过不少新手工程师，一上来就急着配参数，结果项目结构乱成一锅粥。到最后连自己配了什么都找不到。嗯，这节课就是帮你避免这种尴尬。

工作区到底是什么？

工作区（Workspace），说白了就是一个文件夹。它用来存放你的所有配置项目、生成的代码、以及工具的一些中间文件。

我个人习惯把工作区放在一个独立的磁盘分区里，比如D:\Workspace\EB_Tresos。为什么？因为工具在生成代码时会创建大量临时文件，如果放在C盘，很容易把系统盘塞满。我曾经有个同事，项目做到一半，C盘爆红，工具直接崩溃，那叫一个惨。

核心要点：工作区路径不要包含中文、空格或特殊字符。否则编译时你会遇到各种莫名其妙的错误。

创建新MCAL配置项目

创建项目其实很简单，但有几个细节我得提醒你。

  1. 启动EB Tresos，在启动时它会让你选择工作区路径。如果你已经有现成的工作区，直接选；如果没有，新建一个文件夹就行。
  2. 进入主界面后，点击菜单栏的 File → New → AUTOSAR Project。
  3. 弹出一个向导窗口，这里要填几个关键信息：
    • Project Name：项目名称。我建议用芯片型号+功能模块来命名，比如 S32K144_CanDemo。别用 test1、new_project 这种名字，三个月后你自己都看不懂。
    • Project Location：项目存放路径。默认在工作区下，你也可以勾选 Use default location 来指定其他位置。
    • AUTOSAR Version：选择你使用的AUTOSAR版本。比如4.2.2、4.4.0。这里要注意，不同版本的MCAL模块可能不兼容。
    • MCAL Configuration：选择你要配置的MCAL模块。比如Can、Lin、Spi、Adc等。你可以一次性全选，也可以按需选择。
  4. 点击Finish，工具会自动生成项目骨架。

我的小技巧：创建项目时，我习惯先只选一个模块（比如Can），等配置通了再添加其他模块。这样如果出问题，排查范围小很多。

导入现有项目

工作中经常需要接手别人的项目，或者把之前做好的项目搬到新电脑上。这时候就要用到导入功能。

操作步骤：

  1. 点击 File → Import。
  2. 选择 General → Existing Projects into Workspace。
  3. 在 Select root directory 里找到项目文件夹。
  4. 工具会自动识别项目，勾选你要导入的项目，点击Finish。

注意：导入项目时，如果原项目使用了绝对路径（比如引用了某个固定路径的头文件），导入后可能会报错。我曾经遇到过这种情况，折腾了半天才发现是路径问题。所以，建议大家在创建项目时就养成使用相对路径的习惯。

项目结构解析

项目创建完成后，你会看到左侧的Project Explorer里多了一个项目。点开它，里面有很多文件夹和文件。别慌，我来给你拆解一下。

文件夹/文件 作用
config 存放MCAL模块的配置文件（.arxml）。这是你主要操作的地方。
generate 工具生成的代码。包括头文件、源文件、以及一些辅助文件。
output 编译输出文件。比如生成的链接脚本、映射文件等。
src 用户自定义代码。你可以在这里写自己的应用逻辑。
.project Eclipse项目描述文件。记录了项目的基本信息。
.cproject C/C++项目配置。包括编译器选项、链接器设置等。

嗯，这里要特别说一下 config 文件夹。它里面每个模块都有一个独立的 .arxml 文件。比如 Can.arxml、Lin.arxml。双击就能打开图形化配置界面。

你想想看，如果没有这个清晰的结构，几十个模块的配置混在一起，那得多乱？

避坑指南

最后，分享几个我踩过的坑：

  • 不要手动修改生成的文件。比如 generate 文件夹里的代码，每次重新生成都会被覆盖。如果你改了，下次生成就白改了。
  • 定期备份工作区。EB Tresos偶尔会抽风，比如突然崩溃导致项目损坏。我一般每天下班前会把工作区压缩一份，存到网盘里。
  • 版本控制。建议用Git管理项目。但要注意，不要把 generate 和 output 文件夹提交到仓库里，它们太大了，而且每次生成都会变。
